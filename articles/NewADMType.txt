Допустим, у нас есть задание сделать новый справочник. Для того, чтобы переносить на другие стенды записи из этого справочника по отдельности или оптом, нам понадобится новый ADM тип. Для нового ADM типа нам понадобится Интеграционный объект, Контент объект, а также тип данных. IO и CT для нового ADM типа должны начинаться с CADM
+ CADM Product Required Docs IO
+ CADM Product Required Docs CT
+ тип данных (Менеджер внедрения приложений > Проводник типа данных) CADM Product Required Docs
Если ругается "Отсутствует ключ пользователя, который может использоваться для экземпляра компонента интеграции 'Admin Product'.(SBL-EAI-04397)", то нам нужно добавить User Key в Integration Component Key, а также Integration Component Key Fields

Проверять можно в "Менеджер внедрения приложений" - "Проекты для внедрнения". Настроить в нижнем аплете Фильтр для развертывания. Далее "Сеансы внедрения". Если не получается увидеть сформированный xml, значит доступа браузера к папкам на компьютере нет. Попробуйте запуститься с другой машины, например, толстым клиентом.
